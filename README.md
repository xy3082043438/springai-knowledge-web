# SpringAI Knowledge Web (企业级知识库管理台)

> 基于 **Vue 3.5**、**Vite 7.2** 和 **Element Plus** 打造的现代化企业知识库与问答管理前端平台。
> 为 SpringAI Knowledge Server 提供直观的管理界面、流畅的大模型交互体验以及高度密集的“一屏流”数据大屏。

---

## ✨ 核心特性

- 🎨 **极致打磨的 UI/UX**: 采用现代化审美设计（玻璃拟态、光顺动效），全局高度统一了日期时间格式规范与表格交互偏好，提供一流的视觉反馈。
- 📊 **一屏流全维仪表盘**:
  - 精心设计的极简高密度布局，**无需滚动即可一屏全览**所有关键运行指标。
  - 集成 **ECharts 5.6**，实时展示系统状态、知识分布及问答趋势。
  - 支持 **WordCloud** 词云分析，直观穿透知识库核心热点。
- 💬 **沉浸式智能问答**:
  - 仿主流 AI 的聊天界面支持。
  - **精准溯源**: 自动标注引用文档、段落来源，并支持一键定位原始段落。
  - **反馈回路**: 内置满意度评价机制，持续优化知识库质量。
- 📁 **知识库中台**:
  - 文档上传、解析状态追踪。
  - **优雅的沉浸式弹窗预览**，支持原生 PDF 渲染与 Markdown 原文检视。
  - 灵活的 **关联角色配置**: 支持对单个文档或分类设置访问权限。
- 🛡️ **内生 RBAC 管理**:
  - 完整的用户与角色 CRUD。
  - **矩阵式权限分配**: 图形化配置每个角色的接口访问及页面菜单权限。
- ⚙️ **在线参数运维**:
  - 极简表单，图形化调整后端 RAG 参数（检索权重、切分片段大小、模型选择），实时生效。

---

## 🛠 技术架构

- **核心层**: Vue `^3.5.25`, TypeScript `^5.7.2`
- **构建工具**: Vite `^7.2.4` (极致的 HMR 体验)
- **UI 框架**: Element Plus `^2.13.0`
- **状态/路由**: Pinia `^3.0.4`, Vue Router `^4.6.4`
- **可视化**: ECharts `^5.6.0`, ECharts WordCloud
- **网络**: Axios (高度封装，集成 JWT 拦截与统一错误处理)
- **通用工具集**: 独立的日期与文本工具库 (`utils/date.ts`) 规范全局展示逻辑。

## 📁 项目目录导航

```text
src/
  ├── api/          # 分模块定义的后台接口调用 (auth, business, system)
  ├── components/   # 高复用业务组件 (Dashboard 仪表盘组件群、沉浸式预览器等)
  ├── layout/       # 系统骨架 (Sidebar, Navbar, TabsView)
  ├── router/       # 路由配置与基于角色的鉴权守卫
  ├── store/        # Pinia 状态树 (用户 Token, 全局配置)
  ├── styles/       # 系统全局样式与 Sass 混合
  ├── types/        # 强类型 TypeScript 接口定义
  ├── utils/        # 工具类 (权限判断、Request 封装、日期格式化)
  └── views/        # 功能页面 (单屏看板、智能问答、文档中心、系统设置)
```

## 🚀 快速启动

### 环境要求
- **Node.js**: `^20.19.0` 或 `>=22.12.0`

### 1. 安装依赖
```bash
npm install
```

### 2. 本地开发
```bash
npm run dev
```
> 默认运行于: `http://localhost:3000`
> 代理配置: 本地开发请求将自动代理至 `http://localhost:8080/api`。

### 3. 生产打包
```bash
npm run build
```
打包产物位于 `/dist` 目录。

## 🔑 安全与一致性

- **令牌机制**: 登录后 JWT Token 持久化于 `localStorage`，所有请求自动挂载 `Authorization` Header。
- **一致性体验**: 全局复用高度定制化的格式器（如时间处理、统一的 15条/页 分页习惯），确保大规模表单和日志模块的一致性。
- **权限闭环**: 前端路由守卫与后端权限校验双重保障，非授权角色无法看到或调用受限资源。
